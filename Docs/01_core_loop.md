# Core Loop

## Main Loop
1. 准备食材：通过市场购买或战斗掉落获取食材。
2. 摆盘：为战士、猎人、法师三位角色分别编辑饭盒布局。
3. 自动战斗：角色带着饭盒中的食材加成进入自动战斗。
4. 战后结算：获胜后获得金币和掉落食材包。
5. 进入下一节点：在市场或休整节点继续整备饭盒，最终进入 Boss 战。

## Fixed Route
PDF 明确写出的关卡顺序为：

`市场 -> 战斗 -> 休整 -> 战斗 -> 市场 -> 战斗 -> 休整 -> 战斗 -> 市场 -> 战斗 -> 休整 -> 战斗 -> 市场 -> Boss战斗`

玩家没有路线选择权。

## Player Actions
- 切换三名角色的饭盒编辑标签页。
- 从共享仓库中取用食材。
- 旋转、放置、调整食材与饭盒拓展块。
- 在市场购买食材组或饭盒拓展。
- 在休整节点恢复或重排饭盒布局。

## Combat Result Loop
- 若战斗胜利：获得金币、获得对应主题掉落食材包、角色复活并回满生命。
- 若战斗失败：本次冒险结束，从第一关重新开始。
- 若战斗持续超过 `2 分钟`：双方开始等量持续掉血。

## Consumption Rule
- 每个战斗回合结束后，所有在饭盒中已使用的食材都会被消耗掉。
- 玩家只能在库存足够时，一键恢复到上一场战斗前记录的整队方案。
