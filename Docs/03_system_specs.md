# System Specs

## 1. Characters
- 角色固定为：`战士`, `猎人`, `法师`
- 基础数值：
  - 战士：`180 HP / 12 Attack / 1.8 Attack Interval`
  - 猎人：`90 HP / 8 Attack / 0.8 Attack Interval`
  - 法师：`70 HP / 28 Attack / 2.5 Attack Interval`
- 特性：
  - 战士：高血量
  - 猎人：攻击间隔小
  - 法师：攻击高、间隔长

## 2. Bento Box System
- 三个角色各自拥有一个独立饭盒。
- 初始饭盒形状与格子数量相同：`3x3`
- 每个角色的饭盒最终都限制在 `6x8` 空间内。
- 市场中可购买三种角色的饭盒拓展。
- 拓展块之间需要相互连接放置，按边连接。
- 编辑过程中允许临时非法，最终整理结果必须连通。
- 允许自由旋转。
- 允许形成空洞。
- 允许重新拼接。
- 允许改变位置。
- 三个角色通过标签页切换分别编辑。
- 食材仓库共用。
- 可记录上一场战斗前的整队方案，并在战斗后休整中一键恢复；若对应食材不足则不能恢复。

## 3. Food System
- 六类食物：
  - 蔬果
  - 甜品
  - 肉类
  - 饮品
  - 主食
  - 香料
- 食物属性包括：
  - 形状（俄罗斯方块式）
  - 特殊效果
  - 效果触发条件
  - 稀有度
  - 属性加成
  - 金币价值
  - 类别
- 食物允许旋转，旋转后无朝向差异。
- 同一角色饭盒内出现三种同类别食物，则触发此类食物的羁绊效果。
- 触发羁绊效果后，该类别在该饭盒中的全部占格数都会计入效果层数。
- 同一饭盒可以出现多个羁绊组合。

### Bond Effects
- 蔬果：
  - 效果：果酸腐蚀接触的敌怪，即反伤
  - 基础值：凑齐 3 个触发后，反伤 `1`
  - 成长：每多占 `1` 个格子，反伤 `+0.5`
- 甜品：
  - 效果：每隔一段时间给予全队回复
  - 基础值：每 `3` 秒回复 `3 HP`
  - 成长：每多占 `1` 个格子，回复量 `+1`
- 肉类：
  - 效果：血怒，血越低伤越高
  - 触发条件：血量低于 `50%` 开始生效
  - 基础值：每降低 `5%` 生命，攻击提高 `1.5%`
  - 成长：每多占 `2` 个格子，提高手段再 `+1%`，向下取整
- 饮品：
  - 效果：冻结敌方，降低攻速
  - 基础值：降低敌方攻速 `5%`
  - 成长：每多占 `1` 个格子，额外降低 `1.5%`，最多降低 `50%`
- 主食：
  - 效果：敌方血量低于一定下限则斩杀
  - 基础值：敌方血量低于 `5%` 斩杀
  - 成长：每多占 `1` 个格子，提升阈值 `1%`，最高 `20%`
- 香料：
  - 效果：每次攻击附带额外伤害
  - 基础值：每次攻击附加 `1` 点伤害
  - 成长：每多占 `1` 个格子，附加伤害 `+0.5`

## 4. Combat System
- 战斗逻辑采用自走棋式自动战斗。
- 战斗按照攻击间隔、攻击力、血量，再叠加饭盒中食材属性进行实时演算。
- 攻击间隔从开场立刻开始计时，首击需要等一轮。
- 普通攻击默认单体。
- 默认承伤顺序：战士 -> 猎人 -> 法师。
- 怪物特殊技能可具有群体或指定目标效果。
- 怪物特殊属性的目标指定优先级高于默认顺序。
- 若战斗持续超过 `2 分钟`，双方开始等量持续掉血。

### Combat Settlement
- 敌方血量归零：战斗胜利
- 三名主角血量全归零：战斗失败
- 战斗过程中若一个角色死亡，则该角色所有属性和附加效果均不触发。
- 战斗胜利后角色无条件复活并回满生命。

## 5. Monster System
- 每场战斗有且仅有一个怪物。
- 普通怪在探索路线的战斗节点中随机出现，且不会重复出现。
- Boss 固定出现在最后的 Boss 战。
- 怪物基础数值按战斗序号缩放，不按个体固定。

- 玩家无法得知怪物具体属性和具体数值。
- 玩家可以查看系统估算的战斗危险度评级。
- 评级分为：
  - 碾压
  - 稳定
  - 胶着
  - 危险
  - 送死

## 8. Rewards And Consumption
- 普通怪与 Boss 会在特定主题内随机掉落食材包。
- 掉落包中的食材只来自该怪物对应大类。
- 同一掉落包内允许重复出现同种具体食材。
- 普通怪掉落包总价按普通战序号递增。
- 给出的价值锚点为：
  - 第 1 场普通怪：约 `7-9` 金，示例 `8`
  - 第 2 场：`10`
  - 第 3 场：`14`
  - 第 4 场：`18`
  - 第 5 场：`24`
  - 第 6 场：`30`
- 战斗回合结束后，饭盒中已使用的食材全部消耗。

## 11. Market System
- 市场一次展示 `5` 个栏位。
- 花费金币可刷新。
- 刷新成本曲线：
  - `1`
  - `1`
  - `2`
  - `3`
  - `5`
  - `8`
  - `13`
  - `21`
  - 以此类推

### Bento Expansion In Market
- 每个市场格位有 `30%` 概率刷出随机角色饭盒。
- 分布如下：
  - `50%`：`2x2` 饭盒，售价 `16`
  - `25%`：`1x4` 饭盒，售价 `14`
  - `15%`：`2x4` 饭盒，售价 `24`
  - `10%`：`1x1` 饭盒，售价 `6`
- 买到的饭盒本局永久保留，可在后续市场和休整节点任意调整位置。

### Food Purchase In Market
- 每个市场格位有 `70%` 概率刷出随机数量的随机食材。
- 食材组价格 = `食材价值 * 食材数量 * 随机折扣`
- 玩家必须整组买下。
- 每组数量：
  - 普通：`1-6`
  - 稀有：`1-4`
  - 史诗：`1-2`
- 折扣区间：`0.50 - 1.00`
- 折扣越低，出现概率越低，按二次曲线分布。
- 不同市场序号下食材稀有度概率：
  - 市场 1：普通 `70` / 稀有 `30` / 史诗 `0`
  - 市场 2：普通 `50` / 稀有 `40` / 史诗 `10`
  - 市场 3：普通 `25` / 稀有 `60` / 史诗 `15`
  - 市场 4：普通 `20` / 稀有 `50` / 史诗 `30`
