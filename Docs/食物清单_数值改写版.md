# 食物清单（数值改写版）

## 1. 统一数值标尺

本版本食物系统统一采用以下数值等价作为策划配表基准：

- 1金 = 8HP
- 1金 = 1.5ATK
- 1金 = 1.5附伤
- 1金 = 1.7%斩杀线
- 1金 = 10%攻速
- 1金 = 8总治疗
- 1金 = 0.5HPS

说明：

- HP 与总治疗使用同一标尺，便于统一估算续航价值。
- 0.5HPS 按 16 秒有效战斗时长估算，对应约 8 点总治疗。
- 斩杀线默认按普通怪估值；Boss 战建议按 50%效率折算。
- 攻速为强乘区词条，所有带攻速的食物都应谨慎控制面积与副属性。

## 2. combo组建思路

每种种类的食物本身自带的 combo 属性，共 6 种流派。

- 蔬果 -> 香料 -> 肉类 -> 主食 -> 饮品 -> 甜品 -> 蔬果

连携关系：

- 蔬果连接甜品、香料
- 甜品连接蔬果、饮品
- 饮品连接甜品、主食
- 主食连接饮品、肉类
- 肉类连接主食、香料
- 香料连接肉类、蔬果

备注：

- 普通为灰色
- 稀有为蓝色
- 史诗为紫色
- 特殊稀有度保留独立标识接口

## 3. 六大类别定位与数值口径

### 3.1 蔬果

羁绊：果酸反伤（腐蚀接触的敌怪）

定位：

- 主要提供 HP
- 以承伤、反伤、前排续战为核心
- 适合作为战士饭盒的稳定底座

数值换算建议：

- 8HP = 1金
- 反伤总值按总伤害折算，12 点总反伤约等于 1金

配表方向：

- 单品优先给生命、少量防御向词条
- 羁绊收益主要体现在长战斗中的稳定反伤
- 与甜品、香料连携时，偏向拖长战斗换优势

### 3.2 甜品

羁绊：周期回复（每隔一段时间回复）

定位：

- 主要提供血量与治疗
- 是对抗不回满规则的核心续航流派
- 提供容错，帮助队伍跨战保状态

数值换算建议：

- 8 总治疗 = 1金
- 0.5HPS = 1金
- 8HP = 1金

配表方向：

- 单品优先给生命、治疗、少量功能增益
- 羁绊价值可以略高于单件属性价值
- 控制持续时间与触发频率，避免超长战过强

### 3.3 肉类

羁绊：血怒状态（血越低伤越高）

定位：

- 体积较大，形状有棱角
- 主要提供攻击力 ATK
- 是极限反杀与压线通关的核心输出流派

数值换算建议：

- 1.5ATK = 1金
- 低血线增伤建议先折算成等效 ATK 或等效 DPS 再定价

配表方向：

- 单品优先给攻击、少量生命
- 羁绊收益重点放在中低血区间
- 避免在满血时收益过高，否则会压过香料和主食

### 3.4 饮品

羁绊：冻结降速（降低敌方攻速）

定位：

- 形状多为竖长条
- 主要提供攻速
- 是同时强化输出节奏与生存节奏的中轴流派

数值换算建议：

- 10%攻速 = 1金
- 降敌攻速也按等价 10%攻速 = 1金估值，再视稳定性做折扣

配表方向：

- 单品优先给攻速、功能性
- 羁绊偏向团队向收益
- 需要注意与香料的联动会放大高频攻击角色收益

### 3.5 主食

羁绊：血线斩杀（低于阈值直接斩杀）

定位：

- 体积最大，常作为饭盒基础块
- 血量与攻击较为均衡
- 是稳定补足面板并压缩战斗时长的核心流派

数值换算建议：

- 1.7%斩杀线 = 1金
- Boss 战建议按 50%效率折算
- 均衡属性可拆算为 HP 与 ATK 分别计价

配表方向：

- 单品优先给均衡属性
- 羁绊价值强，但应更多体现在收尾而非全程压制
- 大体积形状可适当降低单位面积数值，换取形状价值

### 3.6 香料

羁绊：附加额外伤害（每次攻击附带额外伤害）

定位：

- 体积最小，多为 1x1 或 1x2
- 主要提供附伤与破局词条
- 是补缝、补伤害和做高频连击联动的关键流派

数值换算建议：

- 1.5附伤 = 1金
- 附伤是命中后追加的固定伤害，不吃 ATK 倍率

配表方向：

- 单品优先给附伤、少量功能词条
- 羁绊更适合猎人等高频角色
- 由于形状灵活，允许单位面积价值略低但上手体验更强

## 4. 食物定价原则

单个食物的总价值由四部分组成：

- 基础面板价值
- 羁绊贡献价值
- 形状价值
- 稀有度溢价

建议做表时先按统一数值标尺估算基础价值，再根据形状与稀有度微调。

### 4.1 基础面板价值

- 8HP 记 1 金
- 1.5ATK 记 1 金
- 1.5附伤 记 1 金
- 10%攻速 记 1 金
- 8总治疗 记 1 金
- 1.7%斩杀线 记 1 金

### 4.2 形状价值

同等数值下，形状越灵活，实际价值越高：

- 1x1 最高
- 1x2 / 2x1 次高
- L形 / 折形中等
- 2x2、1x4、2x3 等大块基础块最低

因此大块食物可以用更高的纸面数值交换更低的拼接自由度。

### 4.3 稀有度建议

- 普通：词条单纯，主打一项属性
- 稀有：双属性或一主一副
- 史诗：强主属性 + 明确机制词条

## 5. 配表执行建议

### 5.1 单件食物

建议单件食物的裸价值先控制在：

- 普通：2到3金
- 稀有：4到5金
- 史诗：6到8金

### 5.2 羁绊食物

羁绊食物不应把全部预算都放在裸属性上。建议：

- 60%预算给单件属性
- 40%预算给羁绊贡献和体系价值

### 5.3 功能词条

以下词条建议统一先换算后再组合：

- 生命
- 总治疗 / HPS
- 攻击
- 附伤
- 攻速
- 斩杀线

如果同一食物拥有两个强乘区词条，例如攻速 + 附伤、攻速 + 斩杀，则建议总价值打 0.8 到 0.9 的组合折扣，防止超模。
