# UI Design Rules

## Source Rule
- 界面设计必须优先遵守 `腾讯极限开发策划案.pdf` 中已经明确写出的页面、尺寸、元素和风格要求。
- 若 PDF 未写清楚某个界面细节，必须先询问用户，不能自行决定。
- 引擎迁移到 Godot 后，界面仍按这些设计约束实现，但技术实现方式应遵守 [09_godot_development_mode.md](/D:/2Projects/26.03.28%20Syncark03/09_godot_development_mode.md)。

## Required Screens
- 起始页
  - 标题文本
  - 制作者文本
  - 开始游戏按钮
  - 设置按钮
  - 退出按钮
- 设置页
  - 音量调整
  - 重新开始游戏按钮
- 主编辑页
  - 左侧三角色标签切换
  - 中间饭盒编辑区
  - 底部共享仓库区
  - 右侧开始战斗入口
  - 适合位置显示金币、路线图、战斗力估算等
- 有市场的地方在主编辑页顶部显示市场栏
  - `5` 个商品栏位
  - 刷新按钮
  - 金币显示
  - 食材商品区
  - 饭盒拓展商品区
- 战斗弹出窗口
  - 三名角色状态显示
  - 单个怪物状态显示
  - 角色和怪物手绘风格动画（立绘动态）
  - 实时伤害与回复反馈（数字效果）
  - 战斗结果展示

## Popup Constraints
- 小弹窗：`250 x 100 px`
- 大弹窗：`960 x 500 px`
- 弹窗风格：简单装饰边框 + 右上角关闭按钮

## Art Direction
- 战斗页面风格：手绘极简
- 背景（桌面）参考尺寸：`1920 x 1080`
- 食物图块需要类似俄罗斯方块，具有不同大小和形状
- 饭盒 / 碟子需要有不同式样和大小，通过一组类似 tilemap 的形式组合出来（碗边，碗内角，碗外角，碗中，一格有 9 小格）
- 同类素材可排在同一页导出，其他使用 PNG
- 食物图块统一需要描边

## Layout Priorities
- 角色标签切换必须清晰可见，保证玩家能快速在战士、猎人、法师之间切换。
- 饭盒编辑区必须始终是主视觉区域。
- 仓库区必须清楚区分“库存食材”和“当前饭盒内已放置食材”。
- 市场页必须明确区分“食材商品”和“饭盒拓展商品”。
- 战斗页必须优先保证血量、攻击结果、回复和羁绊反馈的可读性。

## Design Restrictions
- 不允许自行替换成与 PDF 明显不一致的 UI 结构。
- 不允许为了个人偏好增加复杂 HUD、二级菜单或多余页面。
- 不允许在未经用户确认的情况下决定字体、主色、按钮风格、图标体系、动效规范。

## Must Ask The User Before Deciding
- 标题页、设置页、战斗页的最终视觉风格细化
- 字体选择
- 主色与辅色体系
- 按钮样式
- 图标风格
- 页面间切换动效
- 是否需要危险度评级在主编辑页实时显示，以及具体放置位置
