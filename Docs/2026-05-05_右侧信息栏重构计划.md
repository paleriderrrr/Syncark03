# MainEditorScreen 右侧信息栏重构计划

## 最新约束

本次重构按用户最新要求执行，以下约束覆盖之前同主题计划中的旧方案：

- 不再在 `main_editor_screen.tscn` 里直接堆右侧信息栏内部节点。
- 必须创建一个独立的右侧信息板场景，由主场景实例化接入。
- 尽量保证视觉内容都能在 Godot 场景里手动调整，而不是把位置、尺寸、贴图切换规则写死在主界面脚本里。
- 只使用 `Art/UI/NewRightInfo` 中英文命名且本次需要的素材。
- 不使用 `close_button_*` 和 `route-*` 相关素材。

## 选用素材

本次右侧信息板只使用以下四张英文命名素材：

- `Art/UI/NewRightInfo/right_info_front.png`
- `Art/UI/NewRightInfo/right_info_back.png`
- `Art/UI/NewRightInfo/more_info_button_normal.png`
- `Art/UI/NewRightInfo/more_info_button_light.png`

说明：

- `right_info_front.png` 作为正面底板。
- `right_info_back.png` 作为背面底板。
- `more_info_button_normal.png` 与 `more_info_button_light.png` 作为正反两面的按钮贴图状态。
- 悬赏图仍沿用现有怪物悬赏图资源链路，不从 `NewRightInfo` 目录取。

## 目标

将主编辑页右侧怪物信息区重构为一个可复用的独立组件场景：

- 场景名：`Scenes/Components/right_info_board.tscn`
- 脚本名：`Scripts/UI/Components/right_info_board.gd`

主场景只负责：

- 实例化这个组件
- 给组件喂数据
- 决定何时刷新内容

组件自身负责：

- 正反双面显示
- 按钮贴图状态
- 翻面动画
- 正反面节点显隐切换

## 展示方案

### 正面

正面延续当前右侧信息栏的大致布局位置，展示内容为：

- 左上：阶段名
- 左下：怪物悬赏图片
- 右侧从上到下：
  - 怪物名字
  - 危险度
  - 赏金
  - `更多信息` 按钮

### 背面

背面展示：

- 怪物名字
- 怪物悬赏图片
- 由难度曲线乘算之后得到的基础属性
- 怪物技能描述
- 返回正面的按钮

说明：

- 背面的基础属性必须来自当前阶段真实怪物倍率乘算后的结果。
- 不显示怪物模板的未乘算原始值。

## 结构方案

### 新组件场景结构

建议结构如下：

```text
RightInfoBoard
  BoardScale
    FrontFace
      FrontBackground
      FrontStageNameLabel
      FrontWantedPosterRect
      FrontMonsterNameLabel
      FrontRiskLabel
      FrontBountyLabel
      MoreInfoButton
    BackFace
      BackBackground
      BackMonsterNameLabel
      BackWantedPosterRect
      BackStatsLabel
      BackSkillLabel
      BackButton
```

关键原则：

- 正反两面都在组件场景内完成。
- 布局节点都在场景中可视化摆放。
- 主场景不再直接管理正反面内部 label、poster、button 的布局。

### 主场景接入方式

`Scenes/main_editor_screen.tscn` 中：

- 保留 `RightPanel`
- 保留 `SynergyPanel`
- 保留 `ActionButton`
- 用独立实例 `RightInfoBoard` 接管原先的怪物信息展示职责

旧的以下结构退出主职责：

- `NextMonsterPanel`
- `MonsterTooltipPanel`
- `StageInfoPanel`

如果短期为了平滑迁移暂时保留旧节点，也只能处于隐藏和非驱动态，不再参与真实逻辑。

## 可手动调整原则

以下内容必须优先在场景里调，而不是写死在脚本里：

- 正反面背景贴图
- 正反面所有 label 的位置、尺寸、字体大小
- 悬赏图位置与尺寸
- 按钮位置、尺寸、子节点文字
- 信息板整体尺寸
- 正反两面的排布关系

脚本只允许保留这些与表现逻辑强相关的内容：

- 翻面状态机
- 翻面 tween
- 按钮 hover/normal 贴图切换
- 数据写入 label / texture
- 动画过程中的输入锁定

## 数据方案

### 数据来源

继续沿用现有正式数据链路：

- 怪物定义：`MonsterDefinition`
- 下一战怪物预览：`RunState.get_next_monster_summary()`
- 倍率来源：`RunState.get_current_monster_multipliers()`
- 赏金来源：现有战斗奖励曲线
- 危险度来源：现有 `_estimate_risk_label()`

### 组件输入

主界面脚本传给 `RightInfoBoard` 的摘要数据应统一为一个字典，例如：

```text
{
  "stage_name",
  "monster_name",
  "risk_text",
  "bounty_text",
  "stats_text",
  "skill_text"
}
```

另行传入：

- 悬赏图片 `Texture2D`

这样做的原因：

- 组件只消费结果，不耦合 `RunState` 细节。
- 主界面负责把正式数据整理成展示摘要。
- 组件保持可复用和可替换。

## 动画方案

### 目标效果

- 点击正面的 `更多信息` 按钮后，按钮先变淡。
- 信息板执行翻面动画。
- 翻面完成后显示背面。
- 从背面返回正面时，也要走同一套反向翻面流程。

### 实现方式

使用 2D 卡片式翻面方案：

- 以 `BoardScale` 作为动画根节点
- tween `scale.x` 从 `1 -> 0 -> 1`
- 在中点切换 `FrontFace / BackFace` 的显隐

这样做的原因：

- 节点结构简单
- 效果稳定
- 不要求引入额外 3D 体系
- 所有布局仍然由场景可视化编辑

避免的方案：

- 纯 alpha 淡入淡出
- 独立 tooltip 冒层
- 从侧边滑入伪装成翻页

## 实施步骤

### Step 1：创建独立组件

- 新建 `Scenes/Components/right_info_board.tscn`
- 新建 `Scripts/UI/Components/right_info_board.gd`
- 在组件场景内摆好正反面节点
- 把 `NewRightInfo` 的四张素材挂到对应节点上

验收：

- 组件脱离主场景也能单独打开查看结构
- 正反面布局都可在编辑器中手调

### Step 2：接入主场景

- 在 `main_editor_screen.tscn` 中实例化 `RightInfoBoard`
- 主脚本增加 `@onready var right_info_board`
- 原有怪物信息刷新逻辑改为调用组件的 `apply_summary(...)`

验收：

- 主场景已不再依赖 `NextMonsterPanel` 内部节点来驱动展示

### Step 3：迁移数据刷新

- 保持 `RunState.get_next_monster_summary()` 作为怪物预览入口
- 用主脚本整理：
  - 阶段名
  - 怪物名
  - 危险度
  - 赏金
  - 乘算后属性文本
  - 技能描述
- 传入 `RightInfoBoard`

验收：

- 正反两面看到的是同一只怪物的同一套展示语义

### Step 4：移除旧交互依赖

- 取消 `WantedPosterRect` hover 展开 tooltip 的旧逻辑
- `MonsterTooltipPanel` 退出运行职责
- `StageInfoPanel` 不再承担怪物信息展示职责

验收：

- 怪物详细信息只通过翻面进入

### Step 5：更新测试

同步更新：

- `Scripts/Tests/ui_runner.gd`
- `Scripts/Tests/wanted_tooltip_style_runner.gd`

新的测试重点：

- `RightInfoBoard` 存在
- 默认显示正面
- 可翻到背面再翻回正面
- 前后悬赏图同步
- 背面显示属性和技能节点

## 风险与注意事项

- 如果主场景脚本仍直接持有正反面内部 label 节点引用，会破坏“独立组件”边界，必须避免。
- 如果翻面中点切换显隐顺序不对，容易出现前后面同时穿帮。
- 如果任何位置尺寸仍在主界面脚本里硬编码，会违背“场景手动可调优先”的要求。
- 如果保留旧节点却继续被测试或逻辑读取，会造成“看起来换了组件，实际上还在走旧链路”的伪完成状态。

- 正反双面的布局主体可在 Godot 场景编辑器中手动调整。
- 主脚本只负责喂数据，不负责摆正反面内部布局。
- 正面展示：
  - 阶段名
  - 悬赏图
  - 怪物名字
  - 危险度
  - 赏金
  - `更多信息` 按钮
- 点击按钮后，按钮变淡并触发翻面动画。
- 背面展示：
  - 怪物名字
  - 悬赏图
  - 难度曲线乘算后的基础属性
  - 技能描述
- 不使用 `close_button_*` 和 `route-*` 素材。
- 自动化测试已同步更新。
